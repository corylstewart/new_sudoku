Option Model

  1. Purpose

    • Build a web-app that can be used to accurately model an equity option position and all of its greeks.

  2. Goals

    • Create a streaming web-app that uses a range of cached values to create a real time option position.
    • Use a database to store users positions.
    • Provide analytics to show how a position/book would perform in different market conditions.

  3. Features so far

    • Database containing all of a users positions. If the user is not logged in they will have access to the guest account.
    • Although I would like to use real time/delayed market data I feel that it would be a violation of user agreements. So I give the user the most current stock price. From there the user can change the price by hand or use a very simple random stock move generator to see the positions move in "real time".
    • The model that I am using is a very simple Black Scholes implementation. This has many drawbacks the most important of which is that you cannot model dividend paying stocks accurately. Until I come up with a JavaScript model for something better that is going to be a huge limitation.

  4. Future improvements

    • I would like to show P&L graphs for single stocks in a variety of market scenarios.
    • Also, I would like to add an overall book view providing the user all of the greeks from all of the positions in real time.
    • In addition to a book view, I would like to add P&L graphs for the overall book in different market conditions.
    • I have had to limit the size of the tree substantially both to limit instance use and reduce the timeouts.
    • Streaming does not seem like a reasonable option due to the limitations of the system at this time.
    • Add a page for viewing and setting skew versus the implied volatility
    • Find a library or come up with my own implementation of an option pricing model. At this time that is well out of the scope of my own ideas of what this project is about as well as above my mathematics background.

Sudoku

  1. Purpose

    • Create a program that can create and solve valid puzzles of varying degrees of difficulty.

  2. Goals

    • Create algorithms that can solve increasingly harder puzzles and never have to use brute force to solve a valid grid that does not require guessing to solve the problem.
    • Create an algorithm that can create valid Sudoku grids of different difficulties.

  3. Features

    • Ability to solve user generated grids and add them to database of grids.
    • Multiple levels of difficulty based on what techniques are required to solve the puzzle. Using rules such as simple number elimination, pair and triplets, hidden pairs and triplets

  4. Limitations

    • Although while using brute force in addition to my algorithms the program will find a solution. It won't notice if a grid has more than one solution.
    • I would like to add a few more layers of algorithms so that brute force is less used.

  5. Future Improvements

    • Track what puzzles the user has tried and record the times they finished the puzzle in.
    • My old version can generate grids, but I have not implemented that in this version yet.
    • Figure out a better way to generate more difficult grids.

Art Collection

  1. Purpose

    • Create a easy way for my father-in-law to track his art collection.
    • This project is still a work in progress changes are being made daily.

  2. Goals

    • Have simple form for entering information about artwork.
    • Have a simple way to view the collection using different queries.
    • Be able to create reports for export to Excell or MS Access.
    • Send emails to my brother-in-law so that he can track progress.

  3. Features

    • I have built the database and the forms for entering information.
    • Viewing gallery is now searchable by location and medium. Sorting is available on a number of indexes.
    • Personal galleries are only viewable while logged in. When logged in as a guest you see the guest gallery.
    • Art work information is now editable. From the viewing gallery click on an image to get to the edit page.
    • Able to resize images so that the file size fits within a BlobProperty.
    • Seems like I am catching the errors if no file is selected.
    • I have set up an email notification system. I receive alerts when a user sets up an account, and adds photos. I will set it up so that my brother-in-law will gets reports when my father-in-law adds a new piece to his collection.
    • Reports can be generated and uploaded to a Google Drive account. The photos are stored as individual documents, and there is a master spreadsheet containing all of the information about the art.

  4. Future Improvements

    • Figure out what fields are necessary and which are not.
    • Need to fix the spreadsheet generator so that it does a batch change rather then inserting each row.
    • Would like to set up a task queue for generating the reports.
    • Would like the viewing gallery to look a little nicer. Playing with the layout would be nice. But want to nail down the fields before I attempt that.
    • Would be nice to set up the gallery view so that I only hit the DB once, and handle all of the sorting and filtering with javascript.
